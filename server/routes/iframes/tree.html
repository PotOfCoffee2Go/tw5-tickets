To the left is the directory structure of the express server that is the backend to this TiddlyWiki. The `bin` directory contains the node.js applications that can be run using 'npm start' or 'npm run ...' commands referenced in the server's package.json file.

The 'public' directory contains static content delivered by the server. This includes assets such as tiddlywikis, images, database (JSON) files, regular web 'pages', and 'tiddlers' in the TiddlyWiki '.tid' format.

The 'server' directory contains the javascript code for both the server and scripts requested by the browser. The 'server/iframes/scripts' pertain to code specific to <iframe>'s in a tiddler, while 'server/pages/scripts' is for regular web pages.

'server/routes' contains express routes which build dynamic tiddlers and web pages, while 'server/sockets' holds the Socket.IO network interface that allows the server to broadcast tiddlers to TiddlyWiki as server-side tiddler content changes.
